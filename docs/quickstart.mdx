title: Quickstart
description: Start your first workflow run via the REST API
---

## 1. Get an API key

Create a key in the [app UI](https://galaxy-frontend-five.vercel.app/workflows/settings/api-keys) or via `POST /api/v1/api-keys`.

## 2. Find a workflow ID

```bash
curl https://galaxy-backend-kappa.vercel.app/api/v1/workflows \
  -H "Authorization: Bearer $GALAXY_API_KEY"
```

Copy a workflow `id` from the response.

## 3. Start a run

```bash
curl -X POST "https://galaxy-backend-kappa.vercel.app/api/v1/workflows/$WORKFLOW_ID/runs" \
  -H "Authorization: Bearer $GALAXY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{}'
```

Response:

```json
{
  "run": {
    "id": "run_…",
    "workflowId": "wf_…",
    "status": "QUEUED",
    "scope": "FULL",
    "initiator": "API"
  }
}
```

## 4. Poll until complete

```bash
curl "https://galaxy-backend-kappa.vercel.app/api/v1/runs/$RUN_ID" \
  -H "Authorization: Bearer $GALAXY_API_KEY"
```

Wait until `run.status` is `SUCCESS` or `FAILED`. Node outputs are in `nodeRuns`.

## Idempotency

Pass `idempotencyKey` to safely retry run creation without duplicate charges:

```json
{
  "idempotencyKey": "my-unique-request-id-123"
}
```

## Partial runs

Run a single node or selection by passing `targetNodeIds`:

```json
{
  "targetNodeIds": ["node_llm_1"]
}
```

Scope (`FULL`, `SINGLE`, `SELECTION`) is derived automatically from the number of target IDs.
